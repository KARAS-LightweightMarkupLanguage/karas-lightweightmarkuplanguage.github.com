System Requirements

KARAS を導入するシステムが満たすべき条件と、推奨する条件について解説します。

文字エンコ―ディング

KARAS の標準のコンバータでは、文字エンコ―ディングは UTF-8 を対象としています。それ以外のエンコーディングは、考慮していません。したがって、標準のコンバータを用いて KARAS を導入する場合には、システム全体のテキストを UTF-8 に設定してください。

もしも UTF-8 以外のエンコーディングで動作しても、他の異なる環境で、動作することは保障されません。また一部のマルチバイト文字によって不具合を起こす可能性があります。

メディアリンクの処理

KARAS のコンバータは、画像、動画、音楽などのメディアを出力するためのリンクが書かれたとき、それらを任意の出力形式に変換する必要があります。KARAS を導入するシステムの開発者は、メディアのためのリンクを適切に変換する仕組みを実装する必要があります。またその出力結果の例を、KARAS を導入したシステムのユーザに明示することを推奨します。特に、標準のコンバータが出力する結果以外の結果を出力するときは重要なことです。

例えば、標準のコンバータは、 video や audio 要素を使います。あなたはこれを他の Flash などを利用して再生するように変更することができます。ただし、その出力結果の例をユーザに示すことが必要です。

KARAS によって書かれるドキュメントの最終的なフォーマットや、そのドキュメントを読むための環境が分かりません。もしかすると EPUB かもしれないし、PDF かもしれません。ウェブブラウザかもしれませんし、それ以外のブラウザかもしれません。システムの開発者は、それぞれの環境で、メディアが適切に動作するように開発をする必要があります。

セキュリティの問題

KARAS は標準的な出力フォーマットを HTML に設定しています。HTML は Javascript などのスクリプトを埋め込むための仕組みを持っています。また object 要素や、 iframe 要素などの、外部のページやプログラムを実行する仕組みも持っています。Wiki などの不特定多数のユーザがページを編集できるようなシステムでは、悪意のあるユーザが、悪質なスクリプトやタグ、URL を埋め込む危険性があります。KARAS を導入するシステムの開発者は、この点に十分に注意する必要があります。

特定のユーザのみが KARAS を利用するときは、HTML が挿入できる仕組みは大変便利です。しかしながら、不特定多数のユーザが KARAS を利用するようなシステムは、適切なサニタイズ (Sanitize / Sanitization) が必要です。サニタイズに関する技術的な詳細はここでは解説しません。

また、オプションを持つ構文に注意してください。例えばリンク構文や、グループ構文です。特に注意する必要があるのはプラグイン構文です。オプションを利用すると任意のコードを埋め込むことができます。

Plugin に注意する

Plugin の開発や導入には注意が必要です。Plugin の開発者はその Plugin を利用して悪意のあるコードを挿入できる場合に、そのことを示さなければいけません。また配布される Plugin を導入する場合には、その Plugin が悪意のあるコードを出力しないか確認したほうが良いでしょう。また不特定多数のユーザが利用するようなシステムの場合、新しく導入した Plugin によって悪意のあるコードが埋め込まれないか、注意して検証する必要があります。

セキュリティのヒント

KARAS を Wiki などのパブリックなシステムで使うとき、次のようなセキュリティ対策が考えられます。上から順位にセキュリティが強くなります。

  1. ユーザに一部の HTML を挿入させない。
    • 挿入された HTML をエスケープする。
    • 挿入された HTML を削除する。
  2. ユーザに全ての HTML を挿入させない。
    • 挿入された HTML をエスケープする。
    • 挿入された HTML を削除する。
  3. リンクを持ったすべての HTML を削除する。
    • KARAS の構文を使ったリンクの出力も禁止します。
    • a, img, video, object, embed, etc...

またこれらの処理に加えて、Javascript などの危険なテキストの埋め込みを回避する必要があります。
