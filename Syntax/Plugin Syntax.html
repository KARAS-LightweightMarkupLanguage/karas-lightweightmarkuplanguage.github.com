Plugin Syntax

KARAS では、標準的な構文とは別に、特別な構文を持っています。その構文をプラグイン構文と呼びます。プラグイン構文は、単純にマークアップするための以外に、特別な機能を持つことができます。

プラグイン構文は、そのプラグインが導入されたシステム (KARAS) でなければ使うことができません。そのシステムが持つ、特別な構文です。

Description

プラグイン構文はテキストを [[ と ]] で閉じます。プラグイン構文には2つの種類があります。1つはレベル1のコンバート型 (Convert type) です。もう1つはレベル2のアクション型 (Action type) です。

Convert type plugin (Level1)
[[ PluginName :: option1 :: option2... :: markedupText]]
Action type plugin (Level2)
[[[ PluginName :: option1 :: option2...]]]

プラグイン構文は必ずプラグイン名を1つ目のオプションに指定します。オプションは、:: で区切られます。指定したプラグイン名によって、処理の内容が変わります。プラグイン名は大文字であるか小文字であるかを考慮しません。

コンバート型のプラグインは、ある本文テキストを装飾したり、変換したりするために使います。例えば、他のフォーマットで書かれたテキストを変換するために使うことが出来ます。コンバート型のプラグインで本文テキストを指定するときは、必ず、オプションの最後に指定します。

アクション型のプラグインは、そのテキスト全体の内容を使ったテキストを出力するために使います。例えば、目次 TOC (Table Of Contents) のために使います。あるいは、単独で、システムに命令を出すために使うことが出来ます。例えば、コメント欄を挿入したり、ページを非公開にしたりするために使うことが出来ます。

Sample plugin

KARAS の標準のコンバータのセットに含まれるいくつかのプラグインについて解説します。

NowEditiong

NowEditing プラグインは、テキストを出力から削除して、代わりに、 "Now Editing" のテキストを表示するプラグインです。 レベル1 (Convert type) では、オプションとして指定されたテキストを削除します。レベル2 (Action type) では、そのページのすべてのテキストを削除します。

Mark

Mark プラグインは、指定したテキストを Mark タグで閉じるプラグインです。 Mark プラグインはレベル1 (Convert type) しか持ちません。レベル2 (Action type) として実行すると、実行エラーのテキストが得られます。 オプションを2つ指定するとき、1つ目のオプションにマークアップする色を指定することができます。

KARAS は Mark 要素を出力するための構文を持っていません。

TOC (Table Of Contents)

TOC プラグインは、ページに含まれる見出しをリストにして出力するプラグインです。 TOC プラグインはレベル2 (Action type) しか持ちません。レベル1 (Convert type) として実行すると、実行エラーのテキストが得られます。 オプションを1つ指定するとき、1つ目のオプションにリストにする見出しの最小レベルを指定します。2 を指定するとき、レベル1の見出しはリストに含まれません。オプションを2つ指定するとき、リストにする見出しの最大レベルを指定します。3 を指定するとき、レベル 4, 5, 6 の見出しはリストに含まれません。
